HTML

HTML (Hyper Text Markup Language) wird benutzt, um einer Webseite Inhalte zu geben und einem Webbrowser eine Struktur zu verpassen.

Element Content

Der Inhalt (Content) eines HTML Elements ist die Information zwischen dem Öffnungs- (opening tag) und dem Endtag (closing tag)

Beispiel:

Code:

<h1>Codeacademy is awesome!</h1>
Hier ist "Codeacademy is awesome!" der Inhalt

HTML:

Codeacademy is awesom!

Opening Tag

Der Syntax eines "Opening Tag" ist ein «Kleiner als» gefolgt vom Elementnamen und einem "Grösser als".

Beispiel:

<div>

Closing Tag

Ein "Closing Tag" wird benutzt, um das ende eines HTML Elements anzugeben. Der Syntax eines «Closing Tag» ist ein "Kleiner als" gefolgt von einem "Slash", dem Elementnamen und einem "Grösser als".

Beispiel:

<body>
|
</body>

Document Type Declaration

Die erste Linie in einem HTML Dokument muss immer mit <!DOCTYPE html> deviniert sein. Diese Deklaration sagt dem Webbrowser was für eine Art von Dokument zu erwarten ist. In diesem Falle ein HTML5 Dokument.

Beispiel:

<!DOCTYPE html>

HTML Struktur

HTML hat eine Familienbaumstruktur. Ein Element kann Eltern, Grosseltern, Geschwister, Kinder, Enkelkinder, etc. haben.

Beispiel:

<body>
  <div>
    <h1>It's div's child and body's grandschilde</h1>
    <h2>It's h1's sibling</h2>
  </div>
</body>

Comments

Kommentare werden vom Webbrower nicht dargestellt und sind nur im HTML Dokument selbst sichtbar. Sie können einen Abschnitt beschreiben, Gedanken/Ideen festhalten oder Zukünftige Arbeiten aufzeigen. Ein Kommentar kann nur eine einzige Linie benutzen aber auch mehrere.

Beispiel:

<!-- Main Site Content -->
<div>Content</div>

oder

<!--
Comments can be
Multiple lines long.
-->

Whitespace

Als Whitespace werden Tabs, Leerschläge und Zeichenumbrüche bezeichnet. Diese werden in der Darstellung des Webbrowsers ignoriert und dienen hauptsächlich zur Strukturierung des HTML/CSS Dokuments.

Beispiel:

Code:

<p>Test Paragraph</p>

<!-- Der "Whitespace" der über und unter dieser Zeile ist, wird vom Browser ignoriert -->

<p>Another Test Paragraph, this will sit right under the first Paragraph, no extra space between.

HTML:

Test Paragraph

Another Test Paragraph, this will sit right under the first Paragraph, no extra space between.

<html> Element

Das <html> Element sollte direkt nach dem <!DOCTYPE html> eingefügt werden und beinhaltet den ganzen Inhalt und die Struktur des Dokuments.

Beispiel:

<!DOCTYPE html>
<html>
  Inhalt und Struktur der Seite.
</html>

<head> Element

Das <head> Element beinhaltet generelle Informationen über die HTML Seite, die nicht im Browser angezeigt werden. Diese Informationen nennt man auch "Metadaten". Title oder Linkts zu Stylesheets werden hier üblicherweise angegeben.

Beispiel:

<!DOCTYPE html>
<html>
  <head>
    <!-- Metadaten werden hier eingefügt -->
  </head>
</html>

<title> Element

Das <title> Element zeigt den Titel der Seite im Browser/Tab an. Das Element wird nur im <head> Element eingefügt.

Beipiel:

<!DOCTYPE html>
<html>
  <head>
    <title>Titel der HTML Seite</title>
  </html>
</html>

<body> Element

Im <body> Element wird der ganze Inhalt und Struktur der im Browser dargestellten Seite. Es kann nur ein <body> Element pro Dokument existieren.

Beispiel:

<body>
  <h1>Learn to code with CodeAcademy</h1>
</body>

Attribute Name and Values

Ein HTML Attribut besteht aus einem Namen und einen Wert. Es kann auch ein zusätzlicher "syntay" eingefügt werden: name="value" welcher im "opening tag»" zugefügt wird. Dieser wird das "Verhalten" dieses Elements ändern.

Beispiel:

<Elementname name="value">Inhalt</Elementname>

<h1> bis <h6> Heading Elemente

Ein HTML Dokument kann 6 verschiedene Heading Elemente darstellen. Vom grössten <h1> bis zum kleinsten <h6>

Beispiel:

<h1>Breaking News</h2>
<h2>This is the 1st subheading</h2>
<h3>This is the 2nd subheading</h3>
...
<h6>This is the 5th subheading </h6>

<p> Paragraph Element

Dieses Element beinhaltet einen Textblock.

Beispiel:

<p>This is a block of text! Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>

<br> Line Break Element

Ein <br> Line Break Element ist ein Zeilenumbruch, der vom Browser auch dargestellt wird. (Im Gegensatz zum «normalen» Zeilenumbruch, siehe "Whitespace") Dieses Element nutzt nur einen "opening tag" und KEINEN "closing tag".

Beispiel:

Code:

<p>
  A line break haiku. <br>
  Poems are a great use case. <br>
  Oh joy! A line break.
</p>

HTML:

A line break haiku.
Poems are a great use case.
Oh joy! A line break.

<strong> Strong Element

Das <strong> Element setzt den Text zwischen dem "opening tag" und "closing tag" auf bold (Fett). Dies kann zusätzlich zu CSS funktionen über HTML benutzt werden. (Wird aber nur die "normale" bold funktion übernehmen, welche in CSS für 400 steht)

Beispiel:

Code:

<p>This is <strong>important</strong> text!</p>

HTML:

This is important text!

<em> Emphasis Element

Dieses Element wird den Text zwischen dem "opening tag" und "closing tag" auf italicize setzten. Dies kann zusätzlich zu CSS funktionen über HTML benutzt werden.

Beispiel:

Code:

<p>This <em>word</em> will be emphasized in italics.</p>

HTML:

This word will be emphasized in italics.

<div> Div Element

Das <div> Element wird genutzt um "Boxen" (container) zu erstellen, welche das HTML-Dokument in Sektionen aufteilt. Ein <div> Element kann sogenannten "flow content" beinhalten. (z.B. headings, paragraphs, links, images, etc…) Div steht für «division». In Zusammenhang mit CSS und andere Tools (z.B. JavaScript) werden hier auch oft noch class und ID benutzt, um die <div> Container zu benennen, resp. anzusteuern.

Beispiel:

<div>
  <h1>A section of grouped elements</h1>
  <p>Here's some text for the section</p>
</div>
<div>
  <h1>Second section of grouped elements</h1>
  <p>Here's some text</p>
</div>

<span> Span Element

Das <span> Element ist ein sogenannter "inline" Container für Text und kann zum Stylen dieser Gruppe benutzt werden. Jedoch ist das <span> Element ein allgemeiner Container um einen Teil des Textes eines grösseren Text zu trennen. Sollten andere Semantische Elemente zur Verfügung stehen sollten diese Vorrang haben.

Beispiel:

<p><span>This text</span> may be styled differently than the surrounding text.</p>

<li> List Item Element

Das <li> Element generiert eine Liste von "Items". Diese Liste wird in 2 Kategorien unterteilt:

  • Ordered Lists <ol>
  • Unordered Lists <ul>

Beispiel:

Code:

<ol>
  <li>Head east on Prince St</li>
  <li>Turn left on Elisabeth</li>
</ol>

und

<ul>
| <li>Cookies</li>
| <li>Milk</li>
</ul>

HTML:

  1. Head east on Prince St
  2. Turn left on Elisabeth

und

  • Cookies
  • Milk

<ol> Ordered List Element

Das <ol> Element generiert standartmässig eine nummerierte Liste.

Beispiel:

Code>

<ol>
  <li>Preheat oven to 325F</li>
  <li>Drop cookie dougth</li>
  <li>Bake for 15min</li>
</ol>

HTML:

  1. Preheat oven to 325F
  2. Drop cookie dougth
  3. Bake for 15min

<ul> Unordered List Element

Das <ul> Element generiert eine nicht nummerierte Liste. Standartmässig werden hier Punkte statt Nummern als Listenelement dargestellt.

Beispiel:

Code:

<ul>
  <li>Play more music</li>
  <li>Read more Books</li>
</ul>

HTML:

  • Play more music
  • Read more Books

<a> Anchor Element

Das <a> Element wird benutzt um im HTML-Dokument einen Hyperlink zu erstellen. Der Hyperlink kann zu einer anderen Website, einem File auf dem gleichen Server, ein «punk» auf der gleichen Seite oder einer anderen URL über das «href» Attribut weisen. Das «href» Attribut bestimmt das Ziel des Hyperlinks. Text oder auf Bilder die zwischen dem «opening tag» und «closing tag» sind, werden zum Hyperlink.

Beispiel:

<a href="http://www.codeacademy.com">Visit this site</a>

oder

<a href="https://www.codeacademy.com">
  <img src="logo.jpg">Click this image
</a>

<target> Target Attribute

Das <target> Attribut bestimmt, wo der Hyperlink geöffnet wird. Der wird "_blank" wird den Link standartmässig in einem neuen «Tab» öffnen, in einem älteren Browser evtl. auch in einem neuen "Fenster" oder die Einstellungen im Browser wurden so geändert das immer ein neues "Fenster" geöffnet wird.

Beispiel:

<a href="https://google.com" target="_blank">This anchor element links to google and will open in a new tab or window.</a>

Link to a different part of the page #

Das <a> Anchor Element kann Hyperlinks zu verschiedenen Teilen einer Seite führen. Dazu wird im "href" Attribute ein "#"" benutzt, gefolgt von der "ID" des gewünschten Elements.

Beispiel:

<div>
  <p id="id-of-element-to-link-to">A different part of the page!</p>
</div>
<a href="#id-of-element-to-link-to">Take me to a different part of the page</a>

File Path

Ein URL Weg in HTML kann ein "absoluter" Weg sein, wie z.B. eine komplette URL. (Beispiel: https://developer.mozilla.org/en-US/docs/Learn) Oder ein Relativer File Weg der zu einem Lokalem Dokument im gleichen Ordner oder auf dem gleichen Server führt. Z.B. .style.css. Ein Relativer Weg beginnt mit ./ gefolgt vom "file path" zur Lokalen Datei.

Beispiel:

<a href="https://developer.mozilla.org/en-US/docs/web">The URL for this anchor element is an absolute file path.</a>

oder

<a href=".about.html">The URL for this anchor element is a relative file path.</a>

<img> Image Element

Ein <img> Element hinterlegt ein Bild im Dokument. Das "src" Attribut beinhaltet die URL des Bildes und ist demnach Pflicht. Das <img> Element ist ein sogennantes «empty element>, was bedeutet das es keinen "closing tag" besitzt.

Beispiel:

Code:

<img src="resources/images/cutecat.jpg" />

HTML:

"alt" Attribute

Im <img> Element sollte immer auch ein alternativer Text stehen, welcher über das "alt" Attribut erteilt wird. Dieser alternative Text wird nur dann, im Browser dargestellt, wenn das Bild nicht "geladen" werden kann. (Z.B. Das Bildformat wird vom Browser nicht erkannt, das Bild wurde blockiert, ein Fehler in der URL.) Dieser Text wird auch von einen "Screen Reader" vorgelesen und hilft demnach den Inhalt der Seite auch von Sehbeeinträchtigten besser zu verstehen. Ein "alt" Attribut funktionier wie ein "ARIA-Label" hat aber noch zusätzliche Funktionen und sollte daher bei Bildern Vorrang haben.

Wenn ein "alt" Attribut verwendet wird, sollten folgene Sachen beachtet werden:

  1. Generell sollte der "value" vom "alt" Attribut nur das Bild Beschreiben.
  2. Für Bilder die als Hyperlink dienen, sollte das "alt" Attribut auch den Inhalt des Links beschreiben.
  3. Bei einem Bild ohne Informationen (wie ein dekorativer Rahmen) sollte der Inhalt des "alt" Attributs leer sein, das Attribut sollte aber nicht weggelassen werden.
  4. Sollte ein Bild schon beschrieben werden. (z.B. In einem Textabschnitt unter dem Bild selbst) Wird auch hier ein leeres "alt" Attribut verwendet. Die Beschreibung sollte nicht dupliziert werden durch das "alt" Attribut.
  5. Der Inhalt eines "alt" Attributs sollte nicht mehr als 150 Zeichen beinhalten.

Beispiel:

<img src="path/to/image" alt="text describing image" />

<video> Video Element

Ein <video> Element wird im Browser einen "Media Player" anzeigen. Im "src" Attribut wir die URL des Videos hinterlegt. Das hinzufügen des "control" Attribute, gibt dem Video noch die Wiedergabefunktionen wie "play" oder "stop". Ein <video> Element benutzt im Gegensatz des "img" Elements einen "opening tag" und einen "closing tag". Der Inhalt dazwischen ist der "fallback" text (gleich wie ein "alt" Attribut beim "img" Element) und wird nur dargestellt, wenn das Video nicht funktioniert.

Seiten wie YouTube bieten auch Optionen wie "Einbetten/Embed" an mit ganzem Code.

Beispielt:

Code:

<video src="resources/videos/cutecats.mp4" control>
  Video not supportet
< /video>

HTML Attribute

Ein Attribut wird immer im "opening tag" eines Elements verwendet. Durch das hinzufügen eines Attributs kann ein Element spezifischer angepasst werden und auch für weitere Nutzungen anvisiert werden. (Mit Hilfe von "id" und "class") Grundsätzlich sollte zum anvisieren eines Elements zum stylen über CSS ein "class" Attribut dem "id" Attribut bevorzugt werden. (ID's werden in anderen Tools mehr benutzt und sind einzigartig)

Beispiel:

Code:

<p class="my-paragraph" style="color: green;">Here's some text for a paragraph that is being altered by HTML Attributes</p>

oder

<p id="my-paragraph" style="color: red;">Here's some text of a paragraph that is being altered by HTML Attributes</p>

HTML:

Here's some text for a paragraph that is being altered by HTML Attributes

oder

Here's some text of a paragraph that is being altered by HTML Attributes

Unique ID Attributes

Im HTML-Dokument können Elemente mit spezifischen und einzigartigen "id" Attributen versehen werden, welche zur Unterscheidung anderer "gleichen" Elemente dient. Bei gebrauch, können "id" Attribute direkt von CSS oder JavaScript anvisiert werden und diese Elemente zu manipulieren, formatieren oder ihnen spezifische Instruktionen zu geben. Der Inhalt des "id" Attributs sollte nur Buchstaben (a-Z), Nummern (0-9), hyphens (-), underscores (_) und periods(.) enthalten. Genauso werden keine Leerzeichen benutzt.

Beispiel:

<h1 id="A1">Hello World</h1>

<table> Table Element

Im HTML-Dokument wird ein <table> Element benutzt um eine Tabelle mit Reihen (rows) und Spalten (columns) zu erstellen.

Beispiel:

<table>
  <!-- rows and columns will go here -->
</table>

<thead> Table Head Element

Das <thead> Element ist der "header" der Tabelle.

Beispiel:

Code:

<table>
  <thead>
    <tf>
      <th>heading 1</th>
      <th>heading 2</th>
    </tr>
  </thead>
</table>

HTML (Border with CSS):

heading 1 heading 2

<tbody> Table Body Element

Das <tbody> Element ist ein Semantisches Element. Es beinhaltet die ganzen Tabellendaten ausser der "header" und "footer" Daten. Der <tbody> enthält die <tr> "row" Elemente und zeigt an dass das <tr> Element den "body" der Tabelle ausmachen. Ein <table> Element kann nicht <tbody> UND <tr> als direkte "Kinder" haben.

Beispiel:

Code:

<table>
  <tbody>
    <tr>
      <td>row 1</td>
    </tr>
    <tr>
      <td>row 2</td>
    </tr>
  </tbody>
</table>

HTML (Border with CSS):

row 1
row 2

<tfoot> Table Footer Element

Das <tfoot> Element gibt der Tabelle ein "footer" Reihe, welche z.B. zum zusammenfassen einer Reihe genutzt werden kann.

Beispiel:

<table>
  <thead>
    ...
  </thead>
  <tbody>
    ...
  </tbody>
  <tfoot>
    ...
  </tfoot>
</table>

<tr> Table Row Element

Das <tr> Element wird benutzt, um der Tabelle eine Zusätzliche Reihe für Daten <td> oder Headings <th> zu geben.

Beispiel:

<table>
  <tr>
    ...
  </tr>
</table>

<th> Table Heading Element

Das <th> Element wird benutzt um einer Reihe oder Spalte einen Titel zu verpassen. Dieses Element muss in einem <tr> Element sein.

Beispiel:

Code:

<table>
  <tr>
    <th>column</th>
  </tr>
  <tr>
    <td>1</td>
  </tr>
</table>

HTML (Border with CSS):

column
1

<td> Table Data Element

Das <td> Element wird in einer Reihe (<tr>) eingesetzt um Daten in der Tabelle anzuzeigen.

Beispiel:

Code:

<table>
  <tr>
    <td>cell one data</td>
    <td>cell two data</td>
  </tr>
</table>

HTML (Border with CSS):

"rowspan" Attribut

Ähnlich zum "colspan" Attribut wird das "rowspan" Attribut verwendet, um einem Element zu sagen, es solle mehrere Reihen in der Tabelle besetzten. Die "value" des Attributs geht von 1 (default) bis hin zu 65534.

Beispiel:

Code:

<table>
  <tr>
    <th>row 1:</th>
    <td>col 1</td>
    <td>col 2</td>
  </tr>
  <tr>
    <th rowspan="2"> (this row will span 2 rows):</th>
    <td>col 1</td>
    <td>col 2</td>
  </tr>
  <tr>
    <td>col 1</td>
    <td>col 2</td>
  </tr>
  <tr>
    <th>row 3:</th>
    <td>col 1</td>
    <td>col 2</td>
  </tr>
</table>

HTML (Border with CSS):

row 1: col 1 col 2
(this row will span 2 rows): col 1 col 2
col 1 col 2
row 3: col 1 col 2

"colspan" Attribut

Das "colspan" Attribut in einem Header <th> oder in Tabellendaten <td> gibt an wieviele Reihen in einer Spalte benutzt werden. Die «value» geht von 1(default) bis hin zu 1000.

Beispiel:

<table>
  <tr>
    <th>row 1:</th>
    <td>col 1</td>
    <td>col 2</td>
    <td>col 3</td>
  </tr>
  <tr>
    <th>row 2:</th>
    <td colspan="2">col 1 (will span 2 columns)</td>
    <td>col 2</td>
    <td>col 3</td>
  </tr>
</table>

HTML (Border with CSS):

row 1: col 1 col 2 col 3
row 2: col 1 (will span 2 columns) col 2 col 3
